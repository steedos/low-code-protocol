---
title: 开放流程
description: 支持接入不同的流程引擎，实现业务流程自定义。
---

## 核心需求

- 可视化设计业务流程
- 业务对象与流程引擎相关联
- 松耦合，支持挂接不同流程引擎
- 流程同步为元数据

## 数据库设计

### 流程 process

```yml
name: `processApiName`
label: 显示名称
object_name: contracts
engine: activiti-engine/camunda-engine
is_active: true 
entry_criteria: //入口条件公式
```

### 流程版本 process_versions

流程版本记录流程的修订历史，任务在运行时关联到此记录。发起流程时始终取流程最新版。

```yml
_id: xxx
process: `process_id`
version: 2
is_active: true 
schema: `process.activiti.bpmn`
```

## 元数据

元数据下载时只下载流程最新版本。上传时只更新版本。

### 流程 .process.yml

```yml
name: `processApiName`
label: 显示名称
object_name: contracts
engine: activiti-engine
is_active: true 
entry_criteria: //入口条件公式
```

### 流程图 process.activiti.bpmn

基于 activiti 的流程图。在元数据中单独保存此文件，以便在 VS Code 中提供设计器插件。

## 接入 BPM 引擎

使用插件方式接入 BPM 服务。

### 定义 BPM 服务

定义 [Moleculer](https://moleculer.services/docs/0.14/services.html) 微服务

```js
{
    module.exports = {
        name: "activiti-engine",
        settings: {
            serviceEndpoint: "https://activi.services"
        },
        actions: {
            // deploy process
            deploy(ctx) {
                // call activiti api
            },

            // start process
            start(ctx) {
                // call activiti api
            },
            
            beforeCancel(ctx) {
              // 调用 Steedos 流程触发器脚本。
            },
        },
    }
}
```
